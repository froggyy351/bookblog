## プロジェクト概要

技術書管理アプリのバックエンドAPI。
学習した技術書を登録・管理するためのRESTful API。
転職ポートフォリオとして使用予定。

## 技術スタック

| 種別 | 技術 |
|------|------|
| 言語 | Java 24 |
| フレームワーク | Spring Boot 4.0.5 |
| ORM | Spring Data JPA / Hibernate |
| DB | PostgreSQL 17.9 |
| ビルドツール | Maven |
| IDE | IntelliJ IDEA Community |
| テスト | JUnit（予定） |
| コンテナ | Docker（予定） |
| 認証 | Spring Security / JWT（予定） |

## DBテーブル設計

### booksテーブル

| カラム | 型 | 制約 | 備考 |
|--------|-----|------|------|
| id | int | PRIMARY KEY | GENERATED BY DEFAULT AS IDENTITY |
| title | varchar(200) | NOT NULL | |
| author | varchar(200) | NOT NULL | |
| isbn | varchar(20) | NOT NULL | ハイフン含む・10桁or13桁 |
| image | varchar(500) | | 画像URL |
| memo | TEXT | | |
| status | int | NOT NULL | 1:積読 / 2:読書中 / 3:既読 |
| created_at | TIMESTAMP | | |
| updated_at | TIMESTAMP | | @UpdateTimestampで自動更新予定 |

## APIエンドポイント

| メソッド | URL | 説明 |
|----------|-----|------|
| POST | /api/books | 本の登録 |
| GET | /api/books | 本の一覧取得 |
| GET | /api/books/{id} | 本の1件取得 |
| PUT | /api/books/{id} | 本の更新 |
| DELETE | /api/books/{id} | 本の削除 |

## 開発方針・ルール

- コードは自分で調べて書く（AIはヒントのみ）
- コメントアウトされたコードは残さない
- 不要なimportは削除する（Ctrl+Alt+O）
- 例外処理は必ず実装する（ResponseStatusExceptionで404を返す）
- URLは`/api/books`をベースにRESTful設計を維持する
